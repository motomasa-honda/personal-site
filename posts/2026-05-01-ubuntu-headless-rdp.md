## TL;DR
- Ubuntu 24.04標準の「リモートログイン（システムモード）」にはバグがある
- キーリングの競合でヘッドレス起動時に「認証が必要です」で止まる
- 解決策：自動ログイン + Desktop Sharing（ユーザーセッション内）+ キーリングを空パスワード化
- HDMIダミープラグ必須

---

## なぜヘッドレスRDPが必要だったか

Ryzen 9 9950X / RX 7900 XTX 24GBのUbuntu PCをGPU計算ノードとして運用したい。
Mac mini M4からSSH + RDPで操作する構成なので、物理モニター・キーボードは不要。
でもUbuntu 24.04のRDP設定は罠だらけだった。

---

## Ubuntu 24.04標準のリモートログインが使えない理由

Ubuntuの設定UIには「リモートログイン」という項目がある。
これはgnome-remote-desktopのシステムモードで、一見便利そうに見える。

しかし実際に有効化してヘッドレスで起動すると：

1. 起動時に「認証が必要です」ダイアログが表示される
2. モニターがないので操作できない
3. RDPで繋がらないまま詰む

原因はGNOMEキーリングの自動ロック。
システムモードでは起動時にキーリングが開かず、RDPサービスが認証待ちで止まる。

---

## 解決策：ユーザーセッション内のDesktop Sharing

紆余曲折の末に安定した構成がこれ：

| 設定項目 | 値 |
|----------|-----|
| 自動ログイン | **有効** |
| リモートログイン（システムモード） | **無効** |
| 画面共有（Desktop Sharing） | **有効** |
| RDPポート | 3389 または 3390 |
| Loginキーリングのパスワード | **空（なし）** |

ポイントは「システムモード」ではなく「ユーザーセッション内の画面共有」を使うこと。
自動ログインでGNOMEセッションを先に起動させ、そのセッション内でRDPを待ち受ける。

---

## キーリングを空パスワードにする方法

ヘッドレス起動時に「認証が必要です」で止まらないようにするため、
GNOMEキーリングの「Login」キーリングのパスワードを空にする。

設定手順（初回はモニター接続時に実施）：

1. アプリ一覧から「パスワードと鍵」（Seahorse）を起動
2. 左ペインの「Login」を右クリック → 「パスワードを変更」
3. 現在のパスワードを入力 → 新しいパスワードは**空のまま**「続ける」

これで再起動後もモニターなしで自動的にRDPが立ち上がるようになる。
**注意**: この構成は開発用の閉域LAN内マシンに限定した割り切り。外部公開するホストや
本番環境では、キーリングを空にする代わりにVPN経由アクセス+多要素認証などで代替すること。

## ファイアウォール設定

```bash
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 3389/tcp  # RDP（標準ポート）
sudo ufw allow 3390/tcp  # RDP（gnome-remote-desktop設定による）
sudo ufw enable
sudo ufw status
```

---

## HDMIダミープラグが必要な理由

モニターを外した状態だとGPUが画面出力を認識せず、
GNOMEが「ディスプレイなし」と判断して仮想解像度が極端に低くなる（800x600等）。

HDMIダミープラグを挿しておくことで、GNOMEが通常解像度で起動する。
RDP接続時の画面が快適になる。

---

## 最終的な運用状態

- Mac miniのMicrosoft Remote Desktopアプリから `192.0.2.2` に接続
- 物理モニター・キーボードを外した状態で再起動しても自動でRDP接続可能
- SSH（`ssh linuxpc`）とRDPを使い分けてUbuntuを操作

```
Mac mini M4 (192.0.2.1)
    │
    ├── ssh linuxpc          # ターミナル操作
    └── RDP 192.0.2.2      # GUI操作
            │
        Ubuntu 24.04 (192.0.2.2)
        ヘッドレス / 自動ログイン / Desktop Sharing有効
```

---

## 学んだこと

- Ubuntu 24.04の「リモートログイン」（システムモード）はヘッドレス環境では使えない
- 正解は「自動ログイン + ユーザーセッション内のDesktop Sharing」
- キーリングの空パスワード化が再起動後の自動接続に必須
- HDMIダミープラグは地味に重要。解像度問題を防ぐ
- ポート番号はgnome-remote-desktopの設定に従う（3389か3390）
